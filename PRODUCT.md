# Product

## Register

product

## Users

macOS developers and Git power users working in local repositories. They need to understand history, stage precise changes, resolve conflicts, and run branch operations without leaving their current flow or second-guessing what the client will do.

## Product Purpose

git-gud is a focused personal Git client inspired by the strongest interaction patterns in GitKraken. It should make repository state legible, keep common Git workflows fast, and make destructive operations explicit and recoverable. Success means the user can inspect, navigate, and change a repository with the same confidence they would have at the command line.

## Brand Personality

Focused, fast, trustworthy. The interface should feel native to a professional macOS tool: dense where the information earns it, calm around high-risk actions, and direct in its language.

## Anti-references

Do not copy GitKraken branding or reproduce cloud, collaboration, AI, team, and promotional surfaces that are outside this product's local-first scope. Avoid generic SaaS dashboards, decorative gradients, glass effects, oversized cards, novelty controls, and inert chrome that implies unavailable features.

## Design Principles

1. Repository truth comes first. UI state must reflect Git state, not optimistic guesses.
2. Preserve flow. Navigation, search, staging, and history inspection should remain keyboard-accessible and keep spatial context.
3. Make risk visible. Destructive operations identify their scope, explain recovery, and never claim safety without validation.
4. Earn every control. Show functional actions and progressively reveal advanced options instead of displaying dead product areas.
5. Design for real density. The graph, sidebars, and details adapt to constrained windows without sacrificing the primary task.

## Accessibility & Inclusion

Target WCAG 2.2 AA for contrast, keyboard access, focus management, semantic roles, and status announcements. Support reduced motion and do not rely on color alone for selection, status, or graph meaning. Core workflows must remain usable at 200% zoom and at the documented minimum window size.
